# Phase 1 — Feature Parity Audit (Legacy Streamlit vs Current Web)

This document maps the legacy Streamlit business surface to the current Next.js + FastAPI implementation.

## Legend
- **Implemented**: usable in current production web stack (Next.js + FastAPI route wired in main API router).
- **Partial**: some backend/domain support exists, but current Next.js route is placeholder or API router not wired.
- **Missing**: no usable web implementation yet.

## Module parity table

## High-level parity conclusion

- Legacy Streamlit contains the broad business blueprint across dashboard, catalog+stock, sales lifecycle, customers, finance, returns, admin, and settings.
- Current web has **solid auth/session foundation** and **one substantial module (`products-stock`)**.
- Most other modules are currently placeholders on frontend and/or have unmounted backend routers.
